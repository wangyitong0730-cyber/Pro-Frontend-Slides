# Before / After Examples

This file shows the kind of transformation Pro Frontend Slides is designed to produce.

## Related Reading

- [`prompts.md`](prompts.md) — prompt examples for common workflows
- [`showcase.md`](showcase.md) — scenario-driven output goals

## 1. Generic AI Slide Deck → Presentation-First HTML Deck

### Before
- too much text on each slide
- weak hierarchy
- generic fonts and spacing
- every slide feels visually similar
- little sense of pacing during a live presentation

### After
- fewer ideas per slide
- stronger typography and spacing
- visual anchors such as metrics, cards, diagrams, and section slides
- fragments and transitions to control audience attention
- clearer distinction between cover, content, comparison, and closing slides

## 2. Static PPT → Live-Presentable HTML Slides

### Before
- exported PowerPoint layout
- limited motion
- static theme
- difficult to adapt during presenting

### After
- cleaner HTML slide structure
- live theme switching
- presentation controls
- optional drawing mode
- transitions and reveal pacing suited for demos or talks

## 3. Existing HTML Deck → More Polished, More Keynote-Like

### Before
- technically functional slides
- layout inconsistencies
- weak motion language
- text density too high on some slides

### After
- more consistent visual rhythm
- stronger page-to-page pacing
- better use of whitespace
- improved metric and diagram treatment
- more intentional cover and section transitions

## What This Means in Practice

The goal is not to add flashy effects everywhere.
The goal is to make each slide:
- easier to present
- easier to scan
- more memorable
- more visually intentional
